# HeroUI v3 AI-Assisted Development Instructions

## Documentation Reference
Use HeroUI v3 full documentation from: https://v3.heroui.com/llms-full.txt

## Key Principles

### 1. Use tailwind-variants API
- **Always** use `tv()` from `tailwind-variants` for component styling
- Define variants in centralized config files
- Use slots for multi-part components
- Avoid inline Tailwind classes in JSX

### 2. Component Structure
```typescript
import { tv } from "tailwind-variants";

const button = tv({
  base: "font-medium rounded-lg transition-all",
  variants: {
    color: {
      primary: "bg-primary text-white",
      secondary: "bg-secondary text-white",
    },
    size: {
      sm: "h-8 px-3 text-sm",
      md: "h-9 px-4 text-base",
      lg: "h-13 px-6 text-lg",
    },
  },
  compoundVariants: [
    {
      color: "primary",
      size: "lg",
      class: "shadow-lg",
    },
  ],
  defaultVariants: {
    color: "primary",
    size: "md",
  },
});

// Usage: AI-friendly props-based API
<Button color="primary" size="md">Click</Button>
```

### 3. Reluna Design System
- Primary color: #fb6428
- Use design tokens from `config/themes.ts`
- Follow spacing: 4px, 8px, 12px, 16px, 20px
- Border radius: 8px (inputs), 16px (cards), 24px (sections)

### 4. AI Code Generation Rules
- Generate components using **only props**, not inline classes
- Reference variant names from config files
- Use HeroUI's built-in components with custom variants
- Maintain type safety with TypeScript

### 5. MCP Server Available
The project has HeroUI MCP server configured. You can ask:
- "Show me Button component props"
- "Get HeroUI Card examples"
- "What theme variables are available?"

## File Structure
- `config/component-variants.ts` - Centralized variant definitions
- `config/themes.ts` - Reluna design tokens
- `components/` - Reusable components with tv() styling
- `app/` - Pages using variant-based components
